select .. for update

这个是mysql里的一种查询语句格式，翻译是：查找xxx为了更新。这条命令从字面看就是和select查询语句，后面多了一个update。

先说影响：加上了for update的语句的select语句，会在这个查询所在的事务中产生锁行或者是锁表（前提是你这个查询语句已经开启了事务）。

具体会是锁行还是锁表，要看该查询语句是否用到了索引。

  1. 如果该查询用到了索引，这个查询语句只会锁住用到索引的行。比如我用了主键索引：select * from test where id =1 for update。这样我只会锁住id=1的行。
  2. 如果该查询没用到索引，或者索引失效引发的全表扫描，这次查询就会缩表。

for update 就是一个作用，锁。目前我知道的就这些，其他场景网上文章鱼龙混杂，不多赘述，不清楚乱写反而容易错。

mysql的局部索引

常见的mysql数据库里手机号，身份证号，我们经常会简历索引。此外mysql还支持一种叫前缀索引，又叫做局部索引，能够对字段的一部分建立索引。

以手机号为例：

建立收集号前7位的局部索引

上面和常规建立手机号索引基本一样，就是括号里加入了手机号的位数。

  1. 建立局部索引最大的目的就是为了减少索引文件大小，如果手机号前7位区分度已经足够大，就可以用局部索引来代替，可以减少索引文件大小，让每个索引页保存更多的索引值，提高索引查询的速度。
  2. 不过局部索引不能用于order by 和group by ，也不能用于覆盖索引，这是最大的弊端。
