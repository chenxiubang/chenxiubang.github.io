反向代理

Posted on 2021-11-14

反向代理这个词最多出现在nginx。

也是ngxin三特特性：负载均衡，反向代理，动静分离，之一。

但刚学习的时候很多人不知道什么是反向代理。

其实传统的正向代理，也就是代理，在我们访问网站时就经常出现。

比如张三要访问谷歌，但是在国内直接访问国外的域名非常困难。这时候就出现了一个中间代理服务器，用户就通过它，将请求转发给谷歌，实现成功获取资源。这个服务器对于用户是有感知的，用户通过它，访问到了想要的网站。

而nginx的反向代理与此不同，它将真实的服务器藏在自己身后，自己对于用户而言，是一个服务器，用户并无法知道真正的服务器地址。用户将请求交给nginx，由nginx通过负载均衡策略，把请求分发给后台真实的服务集群。再将请求响应给用户。反向代理的nginx对应用户而言，更像一个真实的服务器。

redis基本黑窗口命令1

Posted on 2021-11-09

redis中常用命令：

  1. keys * （查看当前库所有key）
  2. exists （key） （查看是否存在当前名字的key，存在返回1，否则0）
  3. type （key） （查看当前名字key的数据类型，例如返回：string）
  4. del （key) （删除该名字的key，同步，阻塞）
  5. unlink （key）（发送指令后，异步删除该key，系统会提示已经删除）（tip：unlink是解除链接的意思）
  6. select （no) （选择redis的0-15中其中一个数据库）
  7. flashdb （清空当前库所有key）
  8. flushall （删除redis16个库所有的key）
  9. expire （key）（time） （给某个key设置过期时间）
  10. ttl （key) （可以查看该key的剩余过期时间，返回秒，返回-1说明永不过期，返回-2 说明已经过期）
  11. dbsize （返回当前选择库的key的数量）

tips: lop（rpop）k1 count 后面还可以跟一个数字表示弹出元素个数，不过不加具体数目，默认为1

如果把lrange比作substring（），那lindex就是charAt（）。

同理还有获取列表长度的命令 llen :key: , 返回列表长度

此外既然是列表，当然还会有在列表当中插入或者是删除元素的命令

然后还有一个lset，lset k1 index v1 将某个索引下标的值改变成心的值

redis中关于set（集合）的命令

redis中集合的命令都是以s开头的，集合这种数据接口是无序（没有下标），元素不能重复的数据接口。类似于java中的HashSet。

  • 首先是创建/添加的命令：sadd :key: values[…]

    这个是向集合中添加任意数目的元素，返回添加成功的元素个数，（元素不能重复）

  • 集合并集命令：sunion :key:[…]

    这个命令是指定任一个:key:,将它们所有的value组合成一个集合，返回。很简单，类似于mysql的union关键字，命令页非常好记

  • smembers：s+members 用于展示某个：key所有的values值。格式为smembers :key:

  • sismember: s+is+member 。格式 sismember :key: value

    用于判断 某个value值是否属于 该:key:,如果是，就返回1 不是返回0

    注意：sismember命令结尾没有s。

  • spop命令。格式spop :key: count 。用于随机弹出:key:中个数为count的value。这个操作是随机的，所以可以用来抽奖，用count来控制一次弹出的个数

  • scard命令。格式：scard :key: 用来显示某个key的包含的value个数。（其实card在这里翻译的意思很奇怪）

  • srem命令也是用来删除的，不过和spop随机删不一样，srem格式是srem :key: value […]。指定某个key的任意个数的value，将其删除，然后返回成功删除的个数。由于set是没有下标（索引）的，如果想指定删除也的确只有这种指定名字的方式，当然你可以 srem :key: fa!? ->fa 输入一堆没有的value，也不会报错，只不过最后就删除个数返回0，什么也没有删除罢了

  • 此外set的随机命令还有一个就是srandmember可以从集合中取出n个值，但不会像spop一样弹出后就删除

    格式为srandmember :key: count 注意这个地方count是有±的。

    比方说现在的集合是k1:1 2 3 4 5 6

    如果是srandmember k1 2 -> 随机摇出2个是不会重复的-> 1 2 / 1 3 / 2 5

    但如果是srandmember k1 -2 ->随机摇出的2个可能是会重复的 -> 1 1 / 2 2 / 2 5

    如果你说我用srandmember k1 20000这样呢，如果你的count为正数而且远远大于scard k的大小，最后也只会全部返回，不会返回20000个的

  • smove ：将一个集合的一个成员移动到另一个集合当中。格式是smove source destination member

    将source中的member移动到destination中。这个命令倒是不难

  • 集合取交集命令sinter：s + inter.此处inter是交集的缩写。格式很简单

    sinter :key:[…]。会返回所输入集合中的每个集合都存在的元素，然后一起返回

  • 差集命令：sdiff :key: […] 第一个key的集合中，之后k2，k3 ,k4 中都不包含的元素

redis中操作hash类型（适合存放对象的类型）

  1. hset k field value [… ]

    hset现在已经可以像hmset一样对:key:同时放入任意对k v键值对。通过hset成功创建:key:则会返回1。也可以通过hset对已经存在的:key:进行覆盖操作。不过返回值会是0

  2. hget hmget ：hmget 相对于hget 可以一次获取多个field的值，格式 hmget :key: field1 field2 …。然后返回多个值。我觉得能用hmget就没有hget的必要了

  3. hkeys hvals hgetall 这几个的命令都是一样格式：都是命令+:key:

    区别：hkeys获取返回这个k的所有域，hvals获取这个k的所有值，而hgetall返回所有键值对。这个非常像java中hashmap的keyset（），values(),方法

  4. hlen命令，格式 hlen :key: 返回指定key的的hash长度，可以结合strlen来记忆

  5. 我们判断一个key是否存在用的是exists命令，在hash类型的key中，怎么判断某个域是否存在。用的是hexists。格式hexist :key: field。如果这个域存在返回1，否则返回0.

  6. 在string类型的key中我们学过incr incrby命令用于自增数字类型的字符串,在hash类型中同样有类似的命令.

    针对某个域如果是数字类型的格式我们可以用hincrby 或者 hincrbyfloat.两个分别是自增减整型和浮点型的域.

    格式: hincrby(float) :key: field increment 增量可以为正数或者是负数

  7. 在字符串set命令中有一个setnx 用于当某个key is not exists时候才会set成功.在hash类型中有个hsetnx针对hash类型某个域,当这个域已经存在,就不会再set成功导致覆盖这个域.格式hsetnx :key: field value

redis操作Sorted Set（有序集合）的相关

有序集合又叫做zset，他和集合（set）类似，都是属于不可重复的，命令也很相似。不过这种数据结构维护了一种叫做分数（score）的属性，是的这个数据结构有类似于java中TreeSet的自然排序。

下面是一些常见的命令。

  • zadd :key: score member【scores member …】: 和集合添加命令类似，不过开头前缀是z开头的。此外还多维护了一个score的分数属性。返回值是一次成功新添加的元素个数（不包含覆盖）

  • zrange :key: start end [withscores] ，zrevrange :key: start end [withscores]

    先说第一个 ，因为有序集合是有自然顺序的，可以想列表一样更具下标获取范围内的元素。后面的选项参数withscores是可以在显示元素的基础上再显示每个元素的分数。

    有序集合很多命令都有带-rev格式的，其实就是reverse（反转）的缩写，常规命令都是正序的，带rev的都是倒序的。但是实际上有序集合获取范围的命令这个并不是很多，因为这种数据结构更多是为了某个分数区间的，比如下个命令

  • zrangebyscore :key: min max [withscores] 顾名思义，这个就是根据分数区间获取的

  • zscore :key: member 获取具体某个成员在这个有序集合中的分数

  • zcard :key: 有序集合个数

  • zcount :key: min max

    有时候我们不一定想要知道全班的个数，而是想知道某个区间，比如80-90分之间的人数，像这种有序集合获得区间内个数的命令需要用到zcount.

  • zrank :key: member / zrevrank :key: member

    zrank 命令是返回某个成员在有序集合中的下标（这里的下标是从0开始）

  • zrem :key: member [ …]

    又是和zrem 非常像的命令，还是换了个马甲s,返回成功删除的个数

  • zincrby :key: incrememt member :这个是zset命令里的递增命令，可以用在排行版里阅读量的增加。

    递增量可以是正数也可以是负数，也可以是浮点数（不建议，会精度丢失）
